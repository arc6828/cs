# คู่มือการพัฒนาเว็บไซต์หลักสูตรวิทยาการคอมพิวเตอร์ (Vanilla PHP Development Guideline)

ยินดีต้อนรับผู้พัฒนาหลักสูตรวิทยาการคอมพิวเตอร์! เว็บไซต์นี้พัฒนาโดยใช้โครงสร้าง **Vanilla PHP** ร่วมกับระบบการดึงชิ้นส่วนย่อย (Component-Based Inclusion) เพื่อให้ง่ายและสะดวกต่อการบำรุงรักษาโดยเจ้าหน้าที่ โดยมีข้อกำหนดและแนวทางการเขียนโค้ดที่เป็น Best Practice ร่วมกันดังนี้:

---

## 🗂️ 1. โครงสร้างโฟลเดอร์และไฟล์ของระบบ (Project Directory Structure)

ไฟล์แต่ละหมวดหมู่จะจัดแยกแยะเพื่อความเป็นระเบียบและปลอดภัย ป้องกันการนำไฟล์มาวางกระจัดกระจายที่โฟลเดอร์หลัก (Root):

* **`/layouts`**: โฟลเดอร์เก็บชิ้นส่วนร่วมของหน้าเว็บ เช่น ส่วนหัว `<head>` (CSS, SEO) แถบเมนูนำทาง (Navbar) และส่วนท้าย (Footer)
* **`/content`**: ระบบ Router และการแสดงผลเนื้อหาบทความวิชาการ
* **`/css`**, **`/js`**, **`/images`**, **`/fonts`**: สำหรับเก็บไฟล์ Static Assets
* **`/json`**: ที่เก็บฐานข้อมูลสถิตแบบภายในเครื่อง (เช่น ข้อมูลอาจารย์, ข้อมูลคำถาม MBTI)
* **`/uploads`**: สำหรับเก็บไฟล์เอกสารแนบ PDF เช่น คู่มือนักศึกษา
* **`/` (Root Folder)**: เฉพาะไฟล์หน้าเพจแสดงผลหลัก (เช่น `index.php`, `tcas.php`, `contact.php`)

---

## 🎨 2. โครงสร้างไฟล์เพจและสถาปัตยกรรม Layout (Component Injection)

เรายึดถือแนวทาง **"วิธีประกอบชิ้นส่วนย่อย"** โดยปล่อยให้ทุกหน้ามีโครงสร้างแท็ก `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>` เปิดและปิดแบบสมบูรณ์ในตัวเอง เพื่อรองรับการจับคู่แท็กและการจัดรูปแบบอัตโนมัติ (Format Document) ในโปรแกรมแก้ไขโค้ด (IDE เช่น VS Code)

### ตัวอย่างโครงสร้างหน้าเพจมาตรฐาน (สำหรับการสร้างหน้าใหม่ หรือแก้ไขหน้าเดิม)

เมื่อสร้างไฟล์หน้าเว็บใหม่ที่โฟลเดอร์หลัก (Root) ให้ใช้รูปแบบโครงสร้างนี้เป็นมาตรฐานหลัก:

```php
<?php
// 1. (ทางเลือก) ส่วนการประมวลผล PHP ด้านบนสุด และการกำหนดค่า Meta Tags สำหรับ SEO
$og_title = "ชื่อหัวข้อหน้าเว็บ | วิทยาการคอมพิวเตอร์ VRU";
$og_description = "รายละเอียดสั้นๆ สำหรับแชร์ลงโซเชียลมีเดีย";
$og_image = "images/logo.png"; // รูปภาพแสดงเวลาแชร์ลิงก์
?>
<!DOCTYPE html>
<html lang="th">
<head>
    <!-- 2. รวมแท็ก <head>, โหลด CSS Bootstrap 4.5, ฟอนต์ Prompt และ CookieWow -->
    <?php include("layouts/meta.php"); ?>
</head>
<body>
    <!-- 3. รวมสคริปต์เมนูนำทาง Navbar ด้านบน -->
    <?php include("layouts/navbar.php"); ?>

    <!-- 4. ส่วนเนื้อหาเฉพาะของหน้านี้ (เขียนแท็กเปิด-ปิดให้จับคู่กันได้อย่างสมบูรณ์) -->
    <main class="container py-5">
        <h1 class="font-weight-bold mb-4">ชื่อหัวข้อหน้าเว็บ</h1>
        <p class="lead">เนื้อหาและข้อมูลแสดงผลในหน้านี้...</p>
    </main>

    <!-- 5. รวมเนื้อหาส่วนท้ายของเว็บ (ข้อมูลติดต่อหลักสูตรและลิขสิทธิ์) -->
    <?php include("layouts/footer-content.php"); ?>

    <!-- 6. โหลด JavaScript สำหรับการทำงานต่างๆ (เช่น jQuery, Bootstrap JS, AOS) -->
    <?php include("layouts/scripts.php"); ?>
</body>
</html>
```

## ⚡ 3. มาตรฐานการเขียนโค้ดและประสิทธิภาพ (Code & Performance Standards)

### 3.1 การจัดการสำหรับเซิร์ฟเวอร์ที่ทำงานช้า (Handling Slow Server Performance)
เนื่องจากเซิร์ฟเวอร์ของหลักสูตรมีการประมวลผลและการถ่ายโอนข้อมูลที่ค่อนข้างช้า ให้ปฏิบัติตามแนวทางการพัฒนาดังนี้เพื่อเพิ่มความเร็วในการดาวน์โหลดหน้าเว็บ:

1. **ใช้ CDN สำหรับไฟล์สถิต (Use CDN for CSS/JS):**
   * สนับสนุนให้โหลดไลบรารีภายนอก (เช่น Bootstrap, jQuery, FontAwesome, AOS, DayJS) ผ่าน CDN ความเร็วสูง (เช่น `cdnjs`, `jsdelivr`) แทนการโหลดจากเซิร์ฟเวอร์ตัวเอง เพื่อโอนย้ายแบนด์วิดท์ไปทำงานบนโฮสต์คลาวด์ภายนอกที่เร็วกว่า
2. **หลีกเลี่ยงการเชื่อมต่อเน็ตเวิร์กฝั่งเซิร์ฟเวอร์ (No Server-Side Blocking Requests):**
   * การใช้คำสั่ง `file_get_contents("http://...")` เพื่อดึงข้อมูลภายนอกขณะที่ PHP เรนเดอร์หน้าเว็บ จะทำให้เซิร์ฟเวอร์ตัวที่ช้าอยู่แล้วต้องรอข้อมูลเน็ตเวิร์กเพิ่มขึ้นเป็นสองเท่า ส่งผลให้หน้าเว็บโหลดช้ามากหรือค้าง (Timeout)
3. **ย้ายระบบดึงข้อมูลไปทำงานฝั่งผู้ใช้ (Client-Side Rendering / AJAX Fetch):**
   * หากจำเป็นต้องใช้ข้อมูลจากแหล่งภายนอก (เช่น ข้อสอบ MBTI จาก GitHub หรือรายการผลงานวิชาการ) ให้เซิร์ฟเวอร์ส่งคืนเฉพาะหน้าโครงสร้าง HTML เปล่าออกไปทันที จากนั้นใช้สคริปต์ JavaScript `fetch()` บนบราวเซอร์ของเครื่องผู้ใช้งาน เพื่อยิงไปดึงข้อมูลภายนอกและประมวลผลแสดงผลบนหน้าจอเครื่อง Client โดยตรง วิธีนี้ช่วยให้เปิดหน้าเว็บได้ทันทีและลดภาระของเซิร์ฟเวอร์หลักเหลือ 0%

### 3.2 ความปลอดภัยและการตรวจสอบข้อมูล (Input Validation & SQL Injection)
* สำหรับสคริปต์รับค่าส่งฟอร์มทุกตัว ต้องมีการกรองข้อมูลเพื่อป้องกันช่องโหว่ความปลอดภัยเบื้องต้น เช่น การใช้ฟังก์ชัน `htmlspecialchars()` ก่อนแสดงผล และหากมีการเขียนคิวรีฐานข้อมูลในอนาคต ต้องใช้ระบบ **Prepared Statements (PDO)** เท่านั้น

### 3.3 การจัดหน้าและ CSS (Semantic HTML & Bootstrap Grid)
* **หลีกเลี่ยงการเขียนสไตล์ทับ:** หลีกเลี่ยงการใช้แท็กจัดความสวยงามรุ่นเก่า เช่น `<font color="...">` หรือการเขียนสไตล์ดักกว้างๆ เช่น `col-lg-3, li { text-align: center; }` ซึ่งจะไปพังการแสดงผลของหน้าอื่น
* **ใช้ CSS Classes:** ให้ควบคุมความกว้าง ดีไซน์ และการตัดวรรคข้อความผ่าน Utility classes ของ Bootstrap (เช่น `text-white`, `font-weight-bold`, `text-center`, `mb-3`) และหากใช้สไตล์เฉพาะจุด ให้เขียนภายในแท็ก `<style>` ของหน้านั้นๆ หรือจัดเขียนคลาสเฉพาะจุดลงในไฟล์ `css/style.css`

## 📈 5. แนวทางการทำ SEO (Search Engine Optimization) สำหรับนักพัฒนา

เพื่อให้เว็บไซต์หลักสูตรติดอันดับผลการค้นหาที่ดีบน Google (SEO) ผู้พัฒนาต้องปฏิบัติตามแนวทางการเขียนโค้ดดังนี้:

### 5.1 การตั้งค่า Title และ Description เฉพาะหน้า (Unique Meta Tags)
* **กฎเกณฑ์:** ห้ามใช้ Title และ Meta Description ซ้ำกันทุกหน้าเด็ดขาด เพราะ Google จะมองว่าเป็นหน้าซ้ำซาก (Duplicate Content)
* **วิธีทำ:** ในหน้าเว็บหลัก ให้กำหนดตัวแปรภาษาไทยที่กระชับและมีคำสำคัญ (Keywords) เช่น "เรียนวิทยาการคอมพิวเตอร์", "รับสมัคร TCAS" ก่อนเรียกใช้งานไฟล์ `meta.php`:
  ```php
  $og_title = "รับสมัครนักศึกษาใหม่ TCAS 2569 | วิทยาการคอมพิวเตอร์ มรภ.วไลยอลงกรณ์";
  $og_description = "สมัครเรียนปริญญาตรีวิทยาการคอมพิวเตอร์ มรภ.วไลยอลงกรณ์ เรียนต่อสายโปรแกรมเมอร์ นักวิเคราะห์ระบบ ปัญญาประดิษฐ์ ทั้งภาคปกติและเสาร์-อาทิตย์";
  ```

### 5.2 โครงสร้างหัวข้อตามลำดับความสำคัญ (Heading Tags Hierarchy)
* **กฎเกณฑ์:** 
  1. ใน 1 หน้าเว็บ ต้องมีแท็ก `<h1>` **เพียงตัวเดียวเท่านั้น** ซึ่งต้องแสดงชื่อหัวข้อหลักของหน้านั้นๆ (เช่น `<h1>หลักสูตรวิทยาการคอมพิวเตอร์</h1>`)
  2. ใช้ `<h2>` สำหรับหัวข้อย่อย และ `<h3>` สำหรับรายละเอียดภายใต้หัวข้อย่อย ห้ามกระโดดข้ามระดับ เช่น จาก `<h1>` ไปเป็น `<h4>`
* **ตัวอย่างโครงสร้างที่ดี:**
  * `<h1>` รับสมัครนักศึกษา (TCAS)
    * `<h2>` อัตราค่าธรรมเนียมการศึกษา
      * `<h3>` ภาคปกติ (จันทร์-ศุกร์)
      * `<h3>` ภาคพิเศษ (เสาร์-อาทิตย์)

### 5.3 ความเร็วในการเข้าถึงและ Mobile-First Indexing
* Google ใช้คะแนน **Core Web Vitals** (ความเร็วการแสดงผล และความนิ่งของหน้าจอ) ในการจัดอันดับเว็บ และประเมินผลผ่านอุปกรณ์มือถือเป็นหลัก (Mobile-First):
  1. **ลดขนาดรูปภาพ:** ห้ามอัปโหลดรูปภาพต้นฉบับที่มีขนาดใหญ่หลาย Megabytes ขึ้นเซิร์ฟเวอร์โดยตรง ให้แปลงไฟล์ภาพเป็นฟอร์แมต **WebP** และลดขนาดความละเอียดภาพให้พอดีกับการใช้งานจริงก่อน
  2. **หลีกเลี่ยง Cumulative Layout Shift (CLS):** ตรวจสอบว่าแบนเนอร์หรือรูปภาพสไลด์มีมิติสัดส่วนที่ระบุแน่นอน (เช่น มีความสูงขั้นต่ำใน CSS เสมอ) เพื่อป้องกันไม่ให้หน้าเว็บ "กระตุกขยับ" ขณะโหลดรูปเสร็จ ซึ่งทำให้เสียคะแนน SEO

### 5.4 การทำ Structured Data (ข้อมูลโครงสร้าง Schema.org)
* การเพิ่มสคริปต์ **JSON-LD** ในหน้าแนะนำหลักสูตร (`Description.php`, `structure.php`) จะช่วยให้ Google ทราบว่าหน้านี้คือหน้าข้อมูลหลักสูตรปริญญาตรี และแสดงผลลัพธ์เป็น Rich Snippet ที่โดดเด่นในหน้าค้นหา:
  ```html
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "Course",
    "name": "หลักสูตรวิทยาศาสตรบัณฑิต สาขาวิชาวิทยาการคอมพิวเตอร์",
    "description": "มุ่งผลิตบัณฑิตที่มีความรู้ ทักษะทางวิทยาการคอมพิวเตอร์ มีคุณธรรม จริยธรรม...",
    "provider": {
      "@type": "CollegeOrUniversity",
      "name": "มหาวิทยาลัยราชภัฏวไลยอลงกรณ์ ในพระบรมราชูปถัมภ์",
      "sameAs": "http://www.vru.ac.th"
    }
  }
  </script>
  ```
